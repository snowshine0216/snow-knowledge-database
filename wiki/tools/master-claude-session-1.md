## Executive Summary

2-hour live intro pitched at **non-developers** — employees, students, founders, business owners — who already use ChatGPT but have never touched Claude. Mayank frames Claude as three stacked products (Chat / Code / Cowork), teaches one model-picking heuristic (Haiku / Sonnet / Opus → **default to Sonnet**), then argues the real unlock is **Projects + Skills + Connectors**. Most runtime is live demo: workout-plan artifact, India-vs-US buys breakdown, a weekly-report skill composing with web search + docx + Gmail, and a LinkedIn-post skill that meta-analyses its own hooks. Closes with "AI engineers won't be replaced — engineers who don't know AI will" and a "create 10 skills" homework.

## Detailed Chapter Summaries

### 1. Opening
> **Segment**: 00:00–03:00

Live-stream greetings, repeated audio/chat checks, and the first thesis line at 01:23 — "Why Claude? Because it's awesome." Sets a conversational-coaching tone rather than scripted-tutorial tone.

### 2. Why learn Claude now
> **Segment**: 03:00–11:00

Session 1 framing for a free "AI for Everyone" series + cross-promo for a paid 3–4-month "AI Generalist to AI Builder" no-code track starting 2026-05-15. Core argument: the gap between AI *users* and AI *builders* is widening monthly; Session 1 exists to close it for non-coders.

### 3. Anthropic in 2026 — why Claude
> **Segment**: 11:00–14:30

Grounds Claude in 2026 headlines: Anthropic partnership milestones (a "$30 billion" figure is cited) and MCP now under Linux Foundation open governance. Acknowledges that the audience skews ChatGPT and positions the rest of the session as what Claude adds on top.

### 4. Claude ecosystem — Chat / Code / Cowork
> **Segment**: 14:30–17:30

Audience poll: most have not heard of Claude Chat, Claude Code, or Claude Cowork. The speaker insists this three-layer ecosystem map is the first thing to internalise — everything else the session teaches plugs into one of the three.

### 5. Model family — Haiku / Sonnet / Opus
> **Segment**: 17:30–19:00

One heuristic: **default to Sonnet**; switch to **Haiku** for simple tasks run many times a day where token cost matters; switch to **Opus** only when quality matters more than speed and the task is genuinely complex. Analogy — "Einstein is smarter but slower; pick the brain for the question."

### 6. Workout-plan artifact demo
> **Segment**: 19:00–25:00

Live prompt asking Claude for an interactive weekly workout plan. Before generating, Claude asks clarifying questions (fitness level, gym access, output format, session length, target muscles) and only then produces a usable planner. Speaker uses this as the pedagogical hook for why Claude's "question-first" behaviour beats ChatGPT's "produce-immediately" default.

### 7. Claude Mastery Grid 2026 — free resource preview
> **Segment**: 25:00–29:30

Walk-through of a free give-away PDF: **framework + 5 project setups + 5 power skills + 40 domain prompts + quick reference guide**. Role cuts include finance, founder/entrepreneur, marketing, HR/people-ops, sales/BD. Power skills named in the session: meeting notes, weekly report, JD writing.

### 8. What are Skills? — live build
> **Segment**: 29:30–41:00

Defines a Skill as a reusable workflow taught once and auto-invoked on matching tasks. Contrast anchor: today's ChatGPT workflow re-trains the chat every time; with Skills, authoring is one-shot. Live-builds a weekly-report skill, then runs a second "category-based buys breakdown" (India vs. US) through an existing skill to show Claude returns **usable-out-of-the-box** output.

### 9. Skill composition — web search + docx + Gmail
> **Segment**: 41:00–55:00

Adds a recurring instruction to the weekly-report skill: every run searches the web and respects a length cap. Output pipe: Claude-generated report → downloaded as .docx → sent via Gmail connector ("to financial advisor") on user approval. Argument: connectors turn Claude output into real deliverables instead of text-box drafts.

### 10. Connectors, MCP, and LinkedIn-post skill

### 11. Projects — role-specific workspaces
> **Segment**: 70:00–78:00

Switches to the Projects panel: a Project bundles multiple chats plus persistent instructions/files, so every chat inside inherits the same context. Used to explain role-switching — one Project per role, each carrying its own skills and connectors. MCP mentioned as a feature category, not demoed.

## Key Takeaways

1. **Ecosystem before features** — Chat / Code / Cowork is the map; every feature plugs into one.
2. **Default to Sonnet** — one heuristic covers ~95% of use; Haiku for cheap loops, Opus for real complexity.
3. **Let Claude clarify** — one clarified generation beats five unguided retries.
4. **Skills = compounding workflow assets** — author once, amortise across every reuse.
5. **Compose Skills × Connectors** — web search, docx, Gmail, LinkedIn together replace chains of manual steps without code.
6. **Projects = context container** — role-specific workspaces remove the per-chat re-onboarding tax.
7. **Memory is mentioned, not demoed** — expect it in a later session despite the description's promise.
8. **AI literacy is baseline, not bonus** — the "engineers who don't know AI will be replaced" framing is the session's motivational spine.
