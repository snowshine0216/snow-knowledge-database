# 041 — Module 4 Introduction

## Overview

Module 4 focuses on the LLM half of RAG — how LLMs work and how to get the best performance from them within a RAG pipeline.

## Module 4 Roadmap

| Topic | What You'll Learn |
|-------|-----------------|
| Transformer architecture | How LLMs are built and process text |
| LLM calls in code | Constructing and iterating on LLM API calls |
| Grounding techniques | Ensuring LLM responses stay grounded in retrieved context |
| Advanced techniques | Pushing LLM performance limits |
| Practical advice | What actually works in typical RAG projects |
| Programming assignment | Build out a RAG system incorporating Module 4 topics |

## Key Framing

> "The retriever can find and prepare useful information, but at the end of the day, it's the LLM that needs to actually use that information to generate a high-quality response."

The retriever and LLM are complementary: retriever = information finding; LLM = reasoning and generation.

## Key Takeaway
Module 4 completes the picture — after Module 3's retrieval deep-dive, Module 4 covers the generation side: transformer internals, LLM calls, grounding, and advanced prompting.
