# Agent 多模态推理与典型场景

Agent 在多模态任务中的两条核心路径：文字 → 图像语义搜索（CLIP + Milvus），以及图像 → 文字推理（多模态视觉大模型）。涵盖工程实现模式与真实场景案例。

## 两条主线

**文字 → 图像**：用自然语言搜索图像库

```
[离线] 图片 → CLIP 图像编码 → 向量 → 存入 Milvus
[在线] 用户查询 → CLIP 文本编码 → 向量 → Milvus 相似度搜索 → Top-K 图片
```

**图像 → 文字**：用多模态大模型（Qwen-VL / GPT-4V / Claude）对图像进行视觉理解、推理和文字提取。

## CLIP 原理：对比学习

训练时将图文对同时送入模型，语义相近的图文对在向量空间中距离接近（对比损失）。**无需自己训练**，直接使用 OpenAI 预训练权重（如 `ViT-B/32`）。

## Milvus 向量数据库

| 模式 | 适用场景 |
|------|---------|
| Lite | 本地测试 |
| Standalone | 单节点部署（端口 19530 / 9091） |
| Cluster | 生产环境，3 个 Docker 容器 |

核心操作：`insert_data` / `create_index` / `search`。索引操作耗时长，应与用户查询流程分离，在低峰期执行。

## 视觉理解 vs. 传统 OCR

| 对比点 | 传统 OCR | 视觉大模型 |
|--------|----------|-----------|
| 识别依据 | 像素亮度/边缘特征 | 语义理解 |
| 过曝/异常亮度 | 失效 | 可通过场景推理补全 |
| 物体/行为识别 | 需精确标注 | 直接理解动作和关系 |
| 成本 | 低 | 较高，酌情使用 |

## 实际工程场景

- **工地安全**：电焊工作业 vs. 抽烟偷懒 — 传统 OCR 被焊接过曝干扰，视觉模型识别人物姿态
- **无人机巡检**：山火 vs. 汽车尾灯 — 亮度相近导致 OCR 误判，视觉模型理解整体场景语义
- **内容安全审核**：图像/视频中嵌入违规文字（扭曲字、连线干扰）— 视频拆帧逐帧识别

## 工程建议

- 将 CLIP 编码器、Milvus 操作、日志等封装为独立模块，跨项目复用
- 使用 `@log_execution_time` 装饰器监控各步骤耗时，识别冷启动瓶颈
- 视觉大模型成本高，仅在 OCR 确实无法解决时引入
