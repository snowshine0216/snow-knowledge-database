## Key Takeaways

1. **Ecosystem before features.** Before learning Skills or Projects, understand that Claude ships as three products — Chat, Code, Cowork — and everything else plugs into one of them.
2. **Default to Sonnet.** One heuristic covers most users: Sonnet by default, Haiku for cheap high-volume loops, Opus only when the task is complex enough to justify the latency.
3. **Let Claude clarify.** Claude asking a sequence of questions before generating is the feature, not a friction point. One clarified generation beats five unguided retries.
4. **Skills turn one-off prompts into reusable workflows.** Teach once (voice, structure, constraints); every future invocation saves the re-explanation tax.
5. **Compose skills with connectors.** A skill × web-search × Gmail combo replaces a chain of manual steps — generate report, export docx, email it — without writing code.
6. **Projects carry context across sessions.** Role-specific workspaces remove the "re-onboard Claude on every chat" overhead that limits ChatGPT workflows.
7. **Memory is mentioned, not demoed.** Despite the description promising Projects + Skills + Memory, the session covers Projects and Skills deeply but treats Memory only in passing. Expect to learn Memory in a later session.
8. **AI literacy is baseline, not bonus.** The closing argument — "AI engineers won't be replaced; engineers who don't know AI will be" — is the session's main motivational frame, reinforced by the "create 10 skills" homework.

## Practical Pattern

For non-coders trying to lift Claude from "chat tool" to "automated teammate", the session's implicit playbook is:

```
1. Pick a Project for a role (finance advisor, founder, HR, sales, marketing, etc.)
2. Inside the Project, draft 3–10 Skills for tasks you do weekly:
     - weekly report  - meeting notes  - JD writing
     - email reply voice  - LinkedIn post  - status update
3. Wire Connectors: web search, Gmail, LinkedIn.
4. Let Skills compose with Connectors — e.g. "weekly report skill" + web search + "docx export" + Gmail send.
5. Default the executor to Sonnet; upgrade to Opus only when a Skill genuinely demands it.
```

Do not expect a single chat to carry your workflow. Push recurring structure out of the chat into Skills and Projects so the same chat surface can serve multiple roles without re-onboarding.
