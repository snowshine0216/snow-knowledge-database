# 浏览器Action使用指南

## 12306网站专用Action配置

### 核心Action列表

#### navigate - 页面导航
**用途**: 访问12306官网或相关页面
**参数**:
- `url`: 目标URL（必需）
- `snippet`: 操作描述（必需）

**示例**:
```json
{
  "action": "navigate",
  "url": "https://www.12306.cn",
  "snippet": "访问12306官网"
}
```

#### backbone - 页面结构分析
**用途**: 获取12306首页的交互元素结构
**参数**:
- `snippet`: 操作描述（必需）

**返回值**: 包含所有可交互元素的ref映射表
**关键元素ref**:
- `e39`: 出发地输入框 (`fromStationText`)
- `e43`: 到达地输入框 (`toStationText`)  
- `e45`: 日期输入框 (`train_date`)
- `e50`: 查询按钮 (`search_one`)

#### type - 文本输入
**用途**: 在车站输入框中输入城市名称
**参数**:
- `ref`: 元素引用（必需，从backbone获取）
- `text`: 要输入的文本（必需）
- `snippet`: 操作描述（必需）

**示例**:
```json
{
  "action": "type", 
  "ref": "e39",
  "text": "北京",
  "snippet": "输入出发地北京"
}
```

#### press - 键盘操作
**用途**: 执行键盘导航和确认操作
**参数**:
- `key`: 键盘按键（必需）
- `snippet`: 操作描述（必需）

**常用按键**:
- `ArrowDown`: 在下拉列表中选择第一个选项
- `Enter`: 确认当前选择
- `Tab`: 切换到下一个输入框

**示例**:
```json
{
  "action": "press",
  "key": "ArrowDown", 
  "snippet": "选择第一个车站选项"
}
```

#### click - 鼠标点击
**用途**: 点击查询按钮或其他交互元素
**参数**:
- `ref`: 元素引用（必需）
- `snippet`: 操作描述（必需）

**示例**:
```json
{
  "action": "click",
  "ref": "e50",
  "snippet": "点击查询按钮"
}
```

#### readability - 内容提取
**用途**: 从查询结果页面提取结构化内容
**参数**:
- `snippet`: 操作描述（必需）

**返回值**: 清理后的HTML内容，包含车次表格数据

### Action执行顺序

#### 标准查询流程
1. `navigate` → 访问12306官网
2. `backbone` → 获取页面结构
3. `type` (出发地) → 输入出发城市
4. `press` (ArrowDown) → 选择车站
5. `press` (Enter) → 确认选择
6. `type` (到达地) → 输入到达城市  
7. `press` (ArrowDown) → 选择车站
8. `press` (Enter) → 确认选择
9. `click` (查询) → 执行查询
10. `focus_tab` → 切换到结果页面
11. `readability` → 提取结果数据

### 错误处理与重试

#### 常见错误类型
- **Element not found**: ref失效，需要重新执行`backbone`
- **Page not loaded**: 网络超时，需要重试`navigate`
- **Input validation failed**: 车站名称无效，需要修正输入

#### 重试策略
1. **单步重试**: 对失败的单个Action立即重试（最多3次）
2. **流程重试**: 如果连续多个Action失败，重新开始整个流程
3. **智能回退**: 根据错误类型选择合适的回退点

### 性能优化建议

#### 减少API调用
- 复用`backbone`结果，避免重复获取页面结构
- 批量处理相似操作，减少网络延迟影响

#### 异步处理
- 查询执行后立即切换到结果页面，不要等待
- 并行处理多个查询任务（如果需要）

#### 缓存机制
- 缓存常用的车站选择结果
- 缓存近期的查询结果以减少重复查询

### 安全与合规

#### 反爬虫规避
- 控制查询频率（建议间隔2-3秒）
- 避免短时间内大量重复查询
- 使用真实的用户代理和行为模式

#### 数据使用限制
- 仅用于查询目的，不得用于商业用途
- 遵守12306网站的使用条款
- 不得存储或分发查询结果数据

### 调试与监控

#### 日志记录
- 记录每个Action的执行时间和结果
- 记录查询参数和返回结果摘要
- 记录错误信息和重试次数

#### 性能指标
- 平均查询时间
- 成功查询率
- 错误类型分布
- 重试次数统计
