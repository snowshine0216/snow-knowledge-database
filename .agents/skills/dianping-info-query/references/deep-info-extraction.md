# 深度信息提取方法

## 支持的信息类型

### 1. 商户基础信息
- **商户名称**：完整的商户或商圈名称
- **综合评分**：5星制评分（如4.8分）
- **评价数量**：总评论数（如24,285条）
- **商圈位置**：所属商圈（如钟楼/鼓楼人文街区）
- **详细地址**：完整地址信息
- **交通信息**：地铁站、公交站等交通指引
- **营业时间**：每日营业时间段

### 2. 分项评分信息
- **口味评分**：食物口味的专项评分
- **环境评分**：就餐环境的专项评分  
- **服务评分**：服务质量的专项评分
- **食材评分**：食材新鲜度和质量的专项评分（新增）

### 3. 用户标签与高频词
- **标签名称**：用户常用描述词汇
- **提及次数**：该标签被提及的总次数
- **情感倾向**：正面/负面/中性标签分类
- **标签分类**：美食类、服务类、环境类、价格类等

### 4. 推荐菜品
- **菜品名称**：热门推荐菜品列表
- **菜品图片**：菜品展示图片（可选）
- **价格信息**：单品价格（如有）

### 5. 用户评论内容
- **评论摘要**：用户评论的核心观点
- **评论时间**：评论发布日期
- **用户信息**：评论用户（匿名或用户名）
- **评论图片**：用户上传的图片（可选）

## 提取技术方案

### 首选方案：readability API
```javascript
// 使用readability提取结构化内容
const content = use_browser(action="readability")
// 自动包含所有深度信息，格式化良好
```

**优势**：
- 自动处理复杂HTML结构
- 返回干净的文本内容
- 包含完整的评分和标签信息
- 处理图片和多媒体内容

**适用场景**：
- 商户详情页面
- 评论列表页面
- 商圈信息页面

### 备用方案：snapshot + 解析
```javascript
// 获取完整DOM结构
const dom = use_browser(action="snapshot", format="ai")

// 手动解析特定元素
const scoreElement = dom.find('.score')
const tags = dom.querySelectorAll('.tag-item')
```

**优势**：
- 完全控制解析过程
- 可以提取readability无法获取的信息
- 适用于特殊页面结构

**适用场景**：
- readability无法正确提取的页面
- 需要精确控制提取逻辑的场景
- 特殊数据格式的页面

## 商圈查询特殊处理

### 商圈 vs 商户的区别
| 特征 | 商圈 | 商户 |
|------|------|------|
| 评分来源 | 整体商圈评价 | 单一店铺评价 |
| 信息范围 | 包含多个商户 | 仅单个商户 |
| 标签类型 | 区域特色、整体体验 | 具体菜品、服务细节 |
| 地址信息 | 区域范围 | 具体门牌号 |

### 商圈信息提取要点
1. **整体评分**：商圈的综合评分反映整体体验
2. **热门标签**：突出区域特色（如"烟火气十足"、"适合逛吃"）
3. **地理位置**：商圈的大致范围和核心位置
4. **交通便利性**：主要交通方式和站点
5. **热门商户**：商圈内的知名店铺推荐

### 商圈查询示例
```javascript
// 查询西安回民街
use_browser(action="search", query="input[placeholder*='搜索']")
use_browser(action="type", ref="found-ref", text="西安回民街", submit=true)

// 提取商圈信息
const商圈Info = use_browser(action="readability")

// 结果包含：
// - 综合评分: 4.8分
// - 评价数量: 24,285条  
// - 热门标签: ["适合逛吃(3641次)", "烟火气十足(196次)", "网红打卡地聚集(122次)"]
// - 地理位置: 钟楼/鼓楼人文街区
// - 详细地址: 西华门大街与北院门交叉口南200米路西
// - 交通信息: 距地铁钟楼站出入口步行660m
```

## 数据标准化处理

### 用户标签格式化
**原始数据**：
```
周边美食推荐 1815
停车建议 81  
适合逛吃 3641
```

**标准化格式**：
```json
{
  "周边美食推荐": 1815,
  "停车建议": 81,
  "适合逛吃": 3641
}
```

### 评分信息标准化
**原始数据**：
```
★★★★★ 4.8
口味 4.7 环境 4.6 服务 4.5 食材 4.7
```

**标准化格式**：
```json
{
  "overall_score": 4.8,
  "category_scores": {
    "taste": 4.7,
    "environment": 4.6, 
    "service": 4.5,
    "ingredients": 4.7
  }
}
```

## 异常情况处理

### 1. 信息缺失
- **现象**：某些字段为空或不存在
- **处理**：设置默认值或标记为"暂无信息"
- **示例**：营业时间未提供 → "营业时间：暂无信息"

### 2. 数据格式异常  
- **现象**：评分显示为文字而非数字
- **处理**：使用正则表达式提取数字部分
- **示例**："四点八分" → 4.8

### 3. 页面加载不完整
- **现象**：部分内容未加载完成
- **处理**：添加等待时间或重新加载
- **示例**：等待2-3秒后重新提取

### 4. 反爬虫机制
- **现象**：出现验证码或滑块验证
- **处理**：暂停自动化，引导用户手动处理
- **示例**：检测到验证码 → 提示用户完成验证

## 性能优化建议

1. **优先使用readability**：减少手动解析的复杂性
2. **批量提取**：一次操作提取所有需要的信息
3. **缓存结果**：避免重复提取相同页面
4. **异步处理**：对于大量数据采用分批处理
5. **错误重试**：关键信息提取失败时自动重试

## 最佳实践总结

- **readability优先**：90%的场景使用readability即可满足需求
- **动态定位元素**：使用search工具而非硬编码ref
- **标准化输出**：统一数据格式便于后续处理
- **异常处理完善**：覆盖各种边界情况和错误场景
- **商圈特殊处理**：识别并正确处理商圈类型的查询
- **多标签页管理**：确保在正确的页面上执行提取操作
