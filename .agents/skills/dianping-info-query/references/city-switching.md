# 城市切换操作指南

## 概述

本指南详细说明如何在大众点评网站上进行城市切换操作，包括城市选择入口、切换流程、验证方法和常见问题处理。

## 城市切换流程

### 1. 识别当前城市
- **当前位置**: 页面顶部导航栏
- **选择器**: `a.city`
- **验证方法**: 检查元素文本内容，如"上海"、"北京"等

### 2. 访问城市选择页面
- **入口元素**: 城市选择图标
- **选择器**: `.city-select-icon` (img元素)
- **操作方法**: 点击该图标打开城市列表页面

### 3. 城市列表页面结构
- **URL格式**: `https://www.dianping.com/citylist?returl=...`
- **热门城市区域**: 包含"热门城市"标题
- **城市分组**: 按字母A-Z分组显示
- **搜索功能**: 支持通过搜索框查找特定城市

### 4. 城市链接格式
- **标准格式**: `//www.dianping.com/城市拼音`
- **示例**:
  - 北京: `//www.dianping.com/beijing`
  - 上海: `//www.dianping.com/shanghai`
  - 广州: `//www.dianping.com/guangzhou`
  - 深圳: `//www.dianping.com/shenzhen`

### 5. 切换到目标城市
- **定位方法**: 使用`search`工具搜索目标城市名称
- **操作方法**: 点击找到的城市链接
- **等待时间**: 切换后需要等待页面完全加载

### 6. 验证切换成功
- **URL验证**: 检查URL是否包含目标城市拼音
- **城市显示验证**: 检查`a.city`元素是否显示新城市名称
- **页面内容验证**: 确认页面内容已更新为目标城市的数据

## 浏览器工具Action操作示例

### 完整城市切换流程
```javascript
// Step 1: 导航到当前城市页面
use_browser(action="navigate", url="https://www.dianping.com")

// Step 2: 分析页面结构，找到城市选择图标
use_browser(action="backbone")

// Step 3: 点击城市选择图标
use_browser(action="click", ref="city-select-icon-ref")

// Step 4: 在城市列表页面搜索目标城市
use_browser(action="search", query="北京")

// Step 5: 点击找到的北京城市链接
use_browser(action="click", ref="beijing-city-ref")

// Step 6: 验证切换成功
use_browser(action="backbone") // 检查URL和城市显示
```

### 动态城市定位
```javascript
// 使用search工具动态定位城市，避免硬编码
use_browser(action="search", query="目标城市名称")

// 根据返回的ref进行点击操作
use_browser(action="click", ref="found-city-ref")
```

## 常见问题与解决方案

### 1. 城市选择图标未找到
- **原因**: 页面结构变化或元素定位错误
- **解决方案**: 
  - 使用`search`工具搜索"城市"或"切换城市"文本
  - 检查页面是否处于登录状态（可能影响页面结构）
  - 尝试使用文本内容匹配而非CSS选择器

### 2. 城市列表页面加载失败
- **原因**: 网络问题或页面超时
- **解决方案**:
  - 增加等待时间（建议30-60秒）
  - 重试城市切换操作
  - 检查网络连接状态

### 3. 目标城市未在列表中
- **原因**: 城市名称输入错误或该城市不支持
- **解决方案**:
  - 使用城市搜索框精确搜索
  - 确认城市名称的正确拼音格式
  - 检查大众点评是否覆盖该城市

### 4. 切换后页面内容未更新
- **原因**: 缓存问题或页面加载不完整
- **解决方案**:
  - 强制刷新页面
  - 重新执行搜索操作
  - 检查URL是否正确更新

## 最佳实践

### 1. 城市名称标准化
- 使用标准的中文城市名称
- 确保拼音格式正确（全小写，无空格）
- 对于特殊城市名称，提前验证其拼音格式

### 2. 错误处理机制
- 实现重试机制（最多3次）
- 添加超时处理（建议60秒）
- 提供用户友好的错误提示

### 3. 性能优化
- 避免不必要的城市切换操作
- 利用浏览器缓存减少重复加载
- 合理设置等待时间，平衡速度和稳定性

### 4. 兼容性考虑
- 测试不同设备和浏览器的兼容性
- 考虑移动端和桌面端的差异
- 处理不同网络环境下的表现
