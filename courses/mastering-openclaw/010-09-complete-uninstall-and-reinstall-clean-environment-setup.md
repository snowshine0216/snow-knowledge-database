## 什么时候选择"推倒重来"

当遇到以下情况，重装往往比反复排错效率更高：
- 报错不断，多次修复后仍有新问题
- 版本混乱，不确定当前运行哪个版本
- 配置文件损坏，行为异常
- 想从零开始，干净体验

> **核心思想**：就像打扫房间——进入干净环境，比任何"小修小补"更有效。

## 卸载前必须备份

⚠️ **高风险操作警告**：卸载命令会删除所有数据，包括 OpenClaw 的长期记忆、配置、对话记录。

**先备份**：
- OpenClaw 的记忆文件（Memory）
- 自定义 Skills 配置
- API Key 等重要配置
- 任何你需要保留的工作数据

---

## 本地卸载步骤

```bash
# 1. 执行官方卸载命令
openclaw uninstall

# 这会自动清理：
# - OpenClaw 进程
# - 配置文件
# - 数据目录（包括记忆文件）

# 2. 验证清理完成
# 检查是否还有残留进程
```

如果命令无法执行，可以通过包管理器卸载：
- Homebrew 安装的：`brew uninstall openclaw`
- Qcolo 安装的：从应用程序中删除 + 清理数据目录

## 云端卸载步骤

1. **停止/删除云服务器实例**
   - 进入云平台控制台（阿里云/腾讯云等）
   - 找到对应实例，执行"停止"或"销毁"
   - **注意**：检查是否开启了自动续费，避免继续扣费

2. **处理旧 API Key**
   - 到各 AI 平台撤销/删除旧 Key
   - 防止残留 Key 被他人使用，消耗你的额度

---

## 彻底清理的行动清单

| 步骤 | 本地 | 云端 |
|------|------|------|
| 备份数据 | ✅ 备份记忆/配置 | ✅ 导出需要的数据 |
| 停止服务 | `openclaw stop` | 停止实例 |
| 卸载程序 | `openclaw uninstall` | 销毁实例 |
| 清理残留 | 检查进程、删除数据目录 | 关闭自动续费 |
| 吊销 Key | 撤销相关 API Key | 同左 |
| 重新安装 | 按安装教程重来 | 重新购买/部署镜像 |

**最后**：完成清理后，可以重新按照安装流程干净部署，绝大多数奇怪报错都会消失。
