## 为什么要接入通讯渠道

OpenClaw 安装好之后，还差最后一步——接入你日常使用的通讯工具。接入后，AI 才有了"耳朵"和"嘴巴"，能收到你的消息并回复。

**建议**：个人用户优先连接微信（官方直接开放，配置最简单）；企业用户优先连接飞书（企业协作场景更强）。

---

## 接入微信（两种方式）

### 方式一：Qcolo 图形化（最简单）

1. 打开 Qcolo
2. 进入设置 → Channel → 微信
3. 点击连接，弹出二维码
4. 用手机微信扫码确认
5. 显示"连接成功"即完成

连接后，微信里所有发给该账号的消息都会路由到本机 OpenClaw。

### 方式二：命令行官方方式

1. 复制官方连接命令
2. 在终端粘贴运行
3. 等待弹出二维码（同一个 QR 码）
4. 手机扫码确认
5. 通过 OpenClaw Web UI 可查看连接状态

两种方式效果相同，Qcolo 方式更适合新手。

## 接入飞书

飞书需要在**飞书开放平台**创建自定义机器人应用，稍复杂但一次性配置。

### 配置步骤

**1. 进入飞书开放平台**
- 访问飞书开发者后台
- 使用自己的飞书账号登录

**2. 创建自定义应用**
- 企业用户：在"企业应用"下新建
- 个人用户：创建个人应用
- 给应用命名（如"OpenClaw 助手"）

**3. 获取凭证**
- 找到 **App ID** 和 **App Secret**（在应用基本信息页）
- 在"机器人"功能中开启消息能力

**4. 配置权限**
- 开启消息读写权限
- 配置机器人设置（允许接收消息）

**5. 填入 OpenClaw**
- 在 Qcolo 或命令行配置中选择飞书 Channel
- 填入 App ID 和 App Secret
- 保存，测试连接

### 飞书 vs 微信选择建议

| 场景 | 推荐 |
|------|------|
| 个人日常使用 | **微信**（配置最简单） |
| 企业/团队协作 | **飞书**（与企业工具深度集成：会议、文档、日历） |
| 两者都用 | 都接入，根据场景切换 |

---

## 连接后的效果

接入成功后，OpenClaw 以联系人形式出现在微信/飞书中：
- 在微信发消息 → OpenClaw 自动处理并回复
- 在飞书发消息 → OpenClaw 与企业工具联动（会议记录、日程管理等）
- 真正实现 24 小时随时待命的数字员工

> **行动建议**：装好 OpenClaw 后，马上连接至少一个渠道（微信或飞书），开始与它对话，感受真正"为你工作"的 AI。
