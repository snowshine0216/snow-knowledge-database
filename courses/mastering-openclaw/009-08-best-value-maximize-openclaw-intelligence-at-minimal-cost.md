## 模型成本现实分析

很多人觉得"越贵的模型越好"，上来就用 Claude Opus 4.6 导致账单心痛。实际上：

| 模型 | 输入价格 | 输出价格 | 特点 |
|------|---------|---------|------|
| **Claude Opus** | ~$3/M tokens | ~$15/M tokens | 顶级智力，贵 |
| **国产模型（Kimi/DeepSeek）** | ~¥0.75/M tokens | ~¥2/M tokens | 性价比极高 |

**关键洞察**：输出价格比输入贵 5 倍，而 Agent 场景输出量很大，所以模型选择对成本影响极大。

---

## 黄金法则：任务分层调用

> **核心原则：把对的任务交给对的模型，别让高级模型做低级工作**

类比：你不会让专门负责倒垃圾的工人去做精密外科手术——AI 也一样。

**任务分层**：
- **95% 简单任务**：翻译、查询、日程记录、总结 → 国产便宜模型完全胜任
- **5% 复杂任务**：代码调试、复杂推理、创意写作 → 才需要 Claude Opus

这样做的效果：**成本下降 + 响应速度更快**（便宜模型通常延迟更低）。

## 多模型混合配置

在 OpenClaw 配置中设置主/备模型：

```
主模型（Primary）: 用于日常简单任务
  → 推荐：Kimi / DeepSeek（国内直连，便宜）

备用模型（Fallback）: 主模型失败时自动切换
  → 推荐：Claude Opus（复杂任务兜底）
```

**配置方法**：
1. 进入 OpenClaw 模型设置（IWM 配置入口）
2. 设置主模型 → 选国产性价比模型
3. 设置 Fallback 模型 → 选 Claude Opus
4. 系统会根据任务复杂度自动路由，或在主模型报错时切换备用

## 预算上限设置

**强烈建议设置每日/每月 API 预算上限**，原因：

Agent 可能进入错误循环：
> 代码报错 → AI 修复 → 还报错 → 继续修复 → 循环 → 天价账单

**配置方式**：在模型配置中添加预算限制
- 每日上限：5~10 元（基本够用）
- 超出后自动停止调用，发送通知

---

## 推荐组合方案

**极致性价比方案**：

```
日常任务 (95%) → Kimi / DeepSeek
复杂任务 (5%)  → Claude Opus
每日预算上限   → 5~10 元
```

按此配置，每月 AI 成本控制在**咖啡钱**级别（几十元内），而能力不亚于全量使用顶级模型。

**核心思想**：
- 不是买最贵的，而是用最聪明的分层策略
- 任务分类 = 成本分类
- 设置熔断机制，避免失控循环

> [!example]- Answer Guide
> 
> #### Q1 — 任务分层调用黄金法则
> 
> 翻译、查询、日程记录、总结等 95% 的简单任务交给国产便宜模型即可胜任；只有代码调试、复杂推理、创意写作等 5% 的复杂任务才需要 Claude Opus，这样成本下降的同时响应速度也更快。
> 
> #### Q2 — 主模型与 Fallback 角色
> 
> 主模型（Kimi/DeepSeek）负责日常简单任务；当主模型调用失败或报错时，系统自动切换到 Fallback 模型（Claude Opus）兜底处理复杂任务。
> 
> #### Q3 — 极致性价比三要素方案
> 
> 三要素为：日常任务用 Kimi/DeepSeek、复杂任务用 Claude Opus、设置每日 5~10 元预算上限；按此配置每月成本可控制在"咖啡钱"（几十元）级别，核心思想是用最聪明的分层策略替代买最贵的模型，并设置熔断机制防止失控循环。
