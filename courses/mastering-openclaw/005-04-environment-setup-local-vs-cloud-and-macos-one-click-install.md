## 核心问题：OpenClaw 装哪里？

安装前最关键的选择：**本地电脑** 还是 **云端服务器**？这个决策影响：
- 使用体验（系统集成深度）
- 运行成本
- 数据安全与隐私

---

## 本地部署 vs 云端部署对比

| 维度 | 本地部署（MacOS） | 云端部署 |
|------|-----------------|---------|
| **系统集成** | 深度集成 macOS 原生工具，体验最完整 | 无法直接操作本地文件 |
| **数据隐私** | 数据完全在自己电脑，无第三方接触 | 对话数据存在服务商服务器 |
| **成本** | 仅电费，无月租费 | 需按月支付服务器租金 |
| **可用性** | 电脑必须 24 小时开机 | 全天候可用，无需本地设备 |
| **本地资源** | 占用本机计算资源 | 释放个人电脑资源 |
| **功能完整性** | 功能不受限，可操作电脑一切 | 受云端网络限制 |

**本地部署特别适合 Mac 用户**：macOS 的 Shortcuts 等原生工具可与 OpenClaw 无缝集成，真正实现"个人数字员工"的最大价值。

---

## 选择建议：模型才是最大成本

> **核心建议：关注 AI 大模型的价格，而不是平台托管成本**

无论本地还是云端，每月真正的主要成本是调用 AI 大模型的 API 费用（如 Claude Opus）。托管成本相对微不足道。

- **有 Mac 电脑** → 强烈建议本地部署，体验最完整
- **无法保持电脑常开 / 需要 7×24 可用** → 云端部署
- **国内用户** → 本地部署可接入飞书/钉钉/微信等，隐私更有保障

## MacOS 图形化一键安装（Qcolo）

**方式一：Qcolo 图形化安装（推荐新手）**

1. 访问 Qcolo 官网，根据芯片选择下载版本：
   - Apple Silicon（M1/M2/M3）→ 选 ARM 版
   - Intel Mac → 选 x86 版
2. 下载 `.dmg` 文件后双击打开
3. 将应用拖入 Applications 文件夹
4. 打开应用，点击左下角完成 Qcolo 设置
5. 等待安装完成，OpenClaw 即在本地运行

**优点**：安装简单，适合初次体验  
**适用场景**：快速上手、体验核心功能

## 安装后验证：微信连接体验

安装完成后，可连接微信进行验证：

1. 在手机上扫码连接
2. 发送测试消息："今天北京的天气如何，为我把天气记录到备忘录"
3. OpenClaw 会自动完成：查询天气 → 写入备忘录 → 回复结果

**体验变化对比**：
- 之前：通过大模型聊天，感觉像"组合工具"
- 之后：可以把订阅的网站、日程提醒等都托管给 Agent，在微信/飞书/钉钉直接交互，进入工作流——这才是真正"开始为你工作"

---

## 小结

OpenClaw 的安装本身不是难点，关键是**想清楚部署位置的选择**。Mac 用户强烈建议本地部署，可获得最完整的系统集成体验。安装完成后，用两天时间把日常工作习惯迁移进去，会感受到完全不同的 AI 使用体验。
