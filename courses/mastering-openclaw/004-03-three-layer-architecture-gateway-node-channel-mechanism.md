## 三层架构概览

OpenClaw 的核心是三层架构，像一栋大楼的骨架：

| 层级 | 名称 | 职责 |
|------|------|------|
| 第一层 | **Gateway**（网关） | 中央大脑，负责任务调度、记忆管理、消息路由 |
| 第二层 | **Node**（节点） | 执行手臂，在本地实际执行命令和操作 |
| 第三层 | **Channel**（通道） | 连接各种即时通信平台的适配层 |

这三层各司其职，共同构成"随时随地、无处不在"的 AI 体验。

---

## 第一层：Gateway（网关）

Gateway 是整个系统的**大脑和总控中央**，可以把它想象成：
- 一个 24 小时不休息的**接线员**
- 一个**任务调度中央**

**核心职责：**
1. **管理所有连接**：统一接入来自各 Channel 的消息
2. **记忆管理**：维护 AI 的记忆层（昨天发生了什么、对话状态、长期记忆），保证 AI 能记住上下文
3. **任务分发**：决定将任务派给哪一个 AI 员工（Node）处理
4. **安全隔离**：只监听 `127.0.0.1`，不对外暴露——就像银行保险箱放在内部，比暴露在公网的服务安全得多

---

## 第二层：Node（节点）

Node 是**连接大脑与现实世界的手臂**，在本地真正执行操作：

- 浏览网页、执行 shell 命令、操作文件
- 调用外部 API（天气、日历等）
- 甚至控制电脑、关闭窗口

Node 的关键特性是**完全在本地世界运行**，直接操作用户的电脑环境。

---

## 第三层：Channel（通道）

Channel 是**连接各种即时通信平台的适配层**，支持：
- 国际平台：Slack、Telegram 等
- 国内平台：微信、飞书、钉钉、QQ 等

**价值**：用户不需要学习新的工具，在最熟悉的聊天环境里（如微信、飞书）直接和 AI 交互，即可拥有一个超级助手。

---

## 设计哲学：命令行即终极接口

> **"命令行才是智能体设计的终极接口"**

OpenClaw 有别于传统 AI 工具的核心设计理念：

| 传统 AI 工具 | OpenClaw |
|-------------|----------|
| 为每个功能预先构建专用工具 | 只给 AI 最基础的工具（shell 命令行） |
| 功能固定，扩展困难 | AI 自主组合工具完成任意任务 |
| 类比：给工人特定白手套 | 类比：给工人万能工具箱 + 使用说明书 |

这种设计的好处：
- **极其灵活强大**：AI 能执行电脑上任意操作
- **代价**：对 AI 模型的智力（能力）要求极高，必须使用强模型

## 三层协作完整流程示例

**场景**：在钉钉上发消息"查一下北京明天的天气，帮我写到备忘录里"

```
1. Channel 接收钉钉消息
   ↓
2. Gateway 接收消息
   → 加载记忆和当前对话上下文
   → 选择合适的 AI 员工（如"生活助手"）
   ↓
3. AI 员工思考，制定执行计划
   → 第一步：查天气
   → 第二步：写备忘录
   ↓
4. Node 开始执行
   → 调用天气 API → 获取"晴，15-25°C"
   → 将结果反馈给 AI 员工
   → AI 员工生成备忘录内容
   → Node 写入备忘录文件
   ↓
5. Gateway 汇总结果
   → 通过 Channel 回复钉钉
   → 用户看到回复："北京明天晴，15-25°C，已写入备忘录"
```

整个过程只需几秒，却经历了非常多的协作步骤——这正是三层架构的可组合性和可扩展性的体现。

## 本地优先的安全与隐私优势

OpenClaw 将所有处理放在本地的优势：

1. **隐私保护**：主流 APP 的数据都存在厂商服务器，而 OpenClaw 以文件形式存储在本地电脑，不受第三方威胁
2. **数据自主**：记忆文件在本地，可自行管理，随意控制
3. **灵活接入**：可接入任意多的文件，复杂任务接入更多上下文，简单任务保持轻量
4. **记忆分层**：从日常记事 → 对话上下文 → 长期记忆，支持真正长期的工作关系

这个设计目标是：让 AI 成为在所有聊天平台都能陪伴你、同时保持高度安全隐私的数字员工。

> [!example]- Answer Guide
> 
> #### Q1 — Gateway Role and Local Binding
> 
> Gateway 是系统的中央大脑，负责记忆管理、任务分发和消息路由；它只监听 `127.0.0.1` 是安全隔离设计，相当于银行保险箱放在内部而非暴露在公网，大幅降低攻击面。
> 
> #### Q2 — Three-Layer Collaboration Walkthrough
> 
> 消息从钉钉经 Channel 传入 → Gateway 加载记忆并选择合适的 AI 员工 → AI 员工制定计划（查天气 + 写备忘录）→ Node 在本地调用天气 API 并写入备忘录文件 → Gateway 汇总结果后经 Channel 回复钉钉用户。
> 
> #### Q3 — Shell-as-Tool Philosophy Tradeoffs
> 
> 优势：AI 可自主组合工具完成任意任务，极其灵活；代价：不再依赖预构建工具意味着 AI 必须自行理解和拼接底层命令，对模型的推理能力（智力）要求极高，必须使用强模型。
